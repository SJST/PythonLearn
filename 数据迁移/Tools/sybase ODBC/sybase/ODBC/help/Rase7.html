Using LDAP

The Sybase Wire Protocol driver can retrieve connection information either from a data source or from a Lightweight Directory Access Protocol (LDAP) directory. If the driver uses an LDAP directory, the Sybase Wire Protocol data source connects to the LDAP server and retrieves the connection information from the LDAP directory. In this case, the data source’s configuration has only information to connect to the LDAP server and the information in the LDAP directory has the information to connect to the database server. Otherwise, the data source’s configuration has all of the information needed to connect to the database server.

LDAP is a standard protocol for accessing and updating common directory information. Storing connection information centrally in an LDAP directory provides flexibility to make environment changes and reduces the time it takes to reconfigure your infrastructure when a change takes place. For example, if a database must be moved to a different server, you do not have to reconfigure the user applications or the data sources that must now access the new server. Because the connection information is stored in an LDAP directory, you need only update the LDAP directory entries so that the Sybase Wire Protocol driver can connect to the new server.

The Sybase Wire Protocol drivers supports Sybase’s LDAP implementation of an LDAP schema. The driver uses the standard Sybase LDAP connection attribute names for an LDAP directory entry as defined in Sybase’s documentation and configuration files—for example, sybaseAddress.

The values set for the sybaseAddress and sybaseHAServernam connection attributes in a Sybase LDAP directory entry override the values set for the NetworkLibraryName, NetworkAddress, UseSSL, and FailoverNetworkAddress connection attributes defined in the Connect ODBC system information file, but not in a connection string.
